### Phase 1A — 프로필 본문 + 학교/직장 인증

**프로필 편집 화면 구현 (Steps 1–5)**
- 옵션 상수 정의: 얼굴상 8종 / 흡연·음주 4단계 / 종교 5종 / 서울 24개 구 / 태그 26종(6카테고리)
- 기본 정보 입력: 키·거주지역·활동지역·얼굴상·MBTI(4축 이진 칩)·흡연·음주·종교
- 자기소개: 6개 prompt 중 3개 이상 선택 + 텍스트 입력(최대 300자)
- 취향 태그: 카테고리별 칩 그리드, 최대 10개 선택
- `update_profile` RPC(security definer): 전달 필드만 partial update, enum·길이 검증, completeness 재계산(8체크포인트 기준) — 클라이언트가 profiles를 직접 UPDATE 하지 않는 구조 확립

**프로필 미리보기 화면 (Step 6)**
- 학교/직장은 "인증됨" 배지만 표시, 조직명 평문 비노출
- 얼굴상·나이·키·MBTI·지역·태그·자기소개 카드 형식

**학교/직장 인증 (Steps 7–9)**
- `request_org_verification` RPC: 도메인 정책(ac.kr → school, 범용 메일 거부), 6자리 코드 생성 후 sha256 해시 저장, 분당 1회 제한
- 개발 모드 분기: `app.dev_mode` GUC(seed.sql에서 설정)가 true일 때만 평문 코드 반환 — 프로덕션에서는 서버 측에서 차단, 클라이언트 코드에 조건 분기 없음
- `confirm_org_verification` RPC: 해시 비교, 5회 시도 제한, 만료(10분) 체크, 성공 시 status='verified' + reverify_due 6개월 설정
- 앱 UI: 유형 선택 → 이메일 입력 → 코드 입력 → 완료 3단계 플로우

**홈 화면 매칭 준비 안내 (Step 10)**
- 기본정보·자기소개 3개·태그 5개·인증 1개 충족 여부 계산
- 미충족 항목 안내 + 완성도 프로그레스 바 표시
