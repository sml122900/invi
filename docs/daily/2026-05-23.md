### Phase 1A 후속 — `app.dev_mode` GUC → `app_config` 테이블

**문제**
`alter database postgres set "app.dev_mode" = 'true'` 가 로컬 Supabase 의 `postgres` 역할 권한으로 실패. `supabase db reset` 시 seed.sql 실행이 끊김.

**해결**
GUC 대신 `public.app_config (key text primary key, value text)` 테이블 도입. RLS 활성 + security definer 함수만 접근. `request_org_verification` RPC 가 `select value from app_config where key='dev_mode'` 결과를 `coalesce` 로 fallback 처리.
- 로컬: seed.sql 이 `dev_mode='true'` 삽입 → 코드 반환
- 프로덕션: 행 없음 → coalesce false → 코드 미반환

`20260524000001_app_config.sql` 마이그레이션으로 정리.

**Edge Function (`naver-search`) — 네이버 키 격리 (Steps 1–2)**

- JWT 검증으로 로그인 사용자만 호출 허용 (`Authorization: Bearer` 헤더 → `supabase.auth.getUser()`)
- 네이버 지역 검색 API (`openapi.naver.com/v1/search/local.json`) 헤더 `X-Naver-Client-Id/Secret` 를 `Deno.env.get(...)` 으로만 사용 — 클라이언트 번들에 키 미포함
- 응답 정제: `<b>` 태그 strip, `mapx/mapy` 정수(× 1e7) → 위경도 변환, 카테고리 자동 매핑(restaurant/cafe/bar/walk/activity), 원본 네이버 카테고리도 보존
- `.env` 는 `.gitignore` 에 포함되어 키 노출 차단. 메인 `tsconfig` 는 `supabase/functions/**` 를 exclude (Deno 타입 충돌 회피)
